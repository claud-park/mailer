## 각 feature의 Goal (사용자 지정 프로세스, 순서 고정)

0. **superpowers 사용**: plan first → develop → test (brainstorming/writing-plans/TDD 등 해당 스킬 필수 invoke)
1. 해당 feature의 **feature PRD** 생성 → `docs/features/<slug>/PRD.md`
2. PRD 기반 **checkpoint TODO** 생성 → `docs/features/<slug>/TODO.md`
3. 두 문서를 활용해 **If-When-Then 구조의 TC** 생성 → `docs/features/<slug>/TC.md`
4. 모든 설계·개발 결정마다 **선택 이유를 문서화** → `docs/features/<slug>/DECISIONS.md`
5. **/react-best-practices** 스킬로 feasible code structuring
6. **/impeccable** 스킬로 UI 검증, audit pass까지 subagent를 통한 feature 완전 개발
7. **TC로 E2E 테스트** — 전부 통과해야 완료 (실행 비용 정책은 아래 "E2E 실행 비용 정책" 참조)
8. **Obsidian에 기록** → `_obsidian/Projects/ZenMail.md` 체크포인트 추가

## E2E 실행 비용 정책 (사용자 확정, 2026-07-19)

전체 E2E 스위트가 feature마다 영구히 누적되며(현재 250+ 어서션) 매 체크포인트마다 전체를 돌리는 관행이 세션당 토큰 비용을 과도하게 키운다는 문제 제기로 확정:

- **반복 개발 중(iteration)**: 신규 feature의 TC만 스코프를 좁혀 실행한다. `run-tc.mjs`는 각 feature를 자체 fresh user-data-dir 세션(`runXxxSession(page)` 형태, 예: `runMaSession`/`runAltSession`)으로 격리해 두는 기존 관례를 따르면 — 그 세션 함수 하나만 별도 임시 스크립트로 뽑아 실행 가능(파일 앞부분의 helper 함수 선언 전체 + 해당 세션 함수까지를 그대로 복사하고, 마지막에 `launchApp→connectPage→demoLogin→runXxxSession→결과출력` 소형 드라이버만 붙이면 됨 — `PROJECT_DIR`을 zenmail 절대경로로 하드코딩하고 실행 파일은 반드시 `zenmail/` 트리 **안**에 둬야 `node_modules` 해석이 된다). 이 임시 스크립트는 절대 커밋하지 않고 검증 후 즉시 삭제한다.
- **최종 게이트(Goal 7)**: 일반 feature는 전체 스위트 **×1**(1회 클린이면 충분), 대형/위험 feature(플레이키 의심, 대형 리팩터, 여러 뷰·데몬을 동시에 건드리는 변경 등)만 기존대로 **×2 연속** 결정적 확인.
- 새 feature가 자신만의 E2E 세션 함수를 새로 추가할 때는 반드시 fresh user-data-dir 격리 패턴(위 예시)을 따를 것 — 콜드스타트를 검증해야 하는 TC가 있으면 공유 세션 재사용으로는 검증 자체가 불가능하다(new-mail-alerts의 TC-ALT-C1/C2 선례).

## 상시 규약 (사용자 지시, 모든 breaking change마다)

1. `/react-best-practices`로 FE 코드 리뷰 + `/code-review low`
2. 커밋 후 `git@github.com:claud-park/mailer.git` (main)으로 push
3. subagent dispatch 시 모델 명시, 기계적 구현의 하한은 Sonnet (~/.claude/CLAUDE.md 오버라이드)
4. `npm test`(vitest)의 `pretest`가 `better-sqlite3`를 plain-Node ABI로 리빌드해버려 직후 Electron을 띄우면
   `ERR_DLOPEN_FAILED`로 조용히 깨진다(dock 아이콘만 뜨고 창이 없거나, E2E demo-login이 타임아웃) — `npm test`를
   실행했다면 Electron을 다시 띄우기(`npm start`/E2E) 전에 반드시 `npx electron-rebuild -f -w better-sqlite3`를
   먼저 실행한다. 근본 수정(pretest 스크립트를 별도 net-Node 빌드 디렉터리로 분리)은 아직 미착수 백로그.
