# right-reading-pane — DECISIONS

> Goal 4 산출물. D1~D5 전부 사용자 확정 또는 설계 근거 명시 — 미확인 항목 없음.

### D1. 폭 배분 = 고정 비율 40/60 (사용자 확정)
- **이유**: 2026-07-13 브레인스토밍에서 제시된 3옵션(고정 비율 / 드래그 리사이저 / 리스트 고정폭 컬럼) 중 사용자가 고정 비율 40/60을 직접 선택.
- **드래그 리사이저 기각 사유**: 신규 컴포넌트(드래그 핸들·마우스 이벤트 처리)와 폭 값 persist(SQLite settings KV) 상태가 새로 필요해 범위가 증가한다. 이번 feature는 "상하→좌우 전환"만이 목표이므로 YAGNI로 제외.
- **리스트 고정폭 컬럼 기각 사유**: 아바타/발신자만 남기는 좁은 고정폭 컬럼 UI는 기존 ThreadRow의 정보 밀도(발신자·제목·스니펫·라벨)와 근본적으로 충돌 — 별도의 축약 렌더링 체계가 필요해 이번 스코프를 벗어남.
- **후속**: 사용자가 리스트/상세 비율 커스터마이징을 원하면 드래그 리사이저를 별도 feature로 후속 진행.

### D2. 폭 분기 소유자 = ThreadList section 래퍼 (설계 근거 명시)
- **이유**: `ThreadList.tsx`는 현재 fragment(`<>…</>`)를 반환하며 그 안에 BulkActionBanner·SplitTabBar·가상화 리스트가 나란히 들어 있다. `App.tsx`의 `flex-row` 컨테이너에 이 fragment를 그대로 넣으면 세 요소가 각각 flex item으로 취급되어 가로로 흩어진다. `ThreadList` 루트를 `<section>`(flex-col)으로 감싸고 그 section이 폭 분기(`w-2/5` ↔ `flex-1`)를 소유하는 것이 `App.tsx`/`ThreadList` 경계를 깨지 않는 유일한 방법이다.
- **부수 효과(의도된 개선)**: BulkActionBanner·SplitTabBar가 이제 리스트 폭(40%)에만 걸치게 된다 — 상세가 열려 있을 때 배너/탭바가 상세 영역까지 넓게 퍼지지 않는 것이 오히려 좌우 분할 UI에 자연스럽다.

### D3. compact 행에서 라벨 칩 생략 (설계 근거 명시)
- **이유**: 40% 폭에서 2줄 레이아웃에 라벨 칩까지 넣으면 공간이 부족해 스니펫이 잘리거나 사라진다. 상세(ThreadView)가 열려 있는 동안에는 우측 페인 헤더에 해당 스레드의 라벨 칩이 그대로 표시되므로 정보 손실이 없다.
- **대안 기각**: 칩을 축약 아이콘으로 대체하는 안은 신규 시각 언어를 도입해 미니멀리즘 원칙과 충돌하므로 채택하지 않음.

### D4. 행 높이 56↔64 동적 전환 + `virtualizer.measure()` (기술적 근거)
- **이유**: `@tanstack/react-virtual`은 `estimateSize` 함수를 교체하는 것만으로는 이미 측정·캐시된 기존 행 높이가 갱신되지 않는다(라이브러리 특성). compact 전환(`activeThreadId` 변경) 시 `rowHeight`가 바뀌는 `useEffect`에서 `virtualizer.measure()`를 명시적으로 호출해 재측정을 강제한다.
- **infinite scroll 임계값**: 기존 `ROW_HEIGHT * 4` 임계값은 근사치(스크롤 끝 감지용)라 compact 전환 시에도 그대로 두어도 무해 — 정밀도가 필요한 로직이 아니므로 별도 분기를 만들지 않는다(YAGNI).

### D5. `store/mail.ts` 무수정 불변식 (설계 근거 명시)
- **이유**: j/k(`moveSelection`)/Enter(`openThread`)/Esc(`closeThread`)/[/] 내비게이션과 커서 이동 시 자동으로 상세를 갱신하는 "자동 리딩" 로직은 전부 `store/mail.ts` 소유다. 이번 feature는 순수 레이아웃(상하→좌우) 변경이므로 store를 건드릴 이유가 없고, 건드리지 않음으로써 "레이아웃 변경이 내비게이션 회귀를 만들 수 없는 구조"를 보장한다.
- **검증 방법**: TC-RP-A4(j 2회 이동 시 우측 페인이 따라 갱신)로 자동 리딩 무회귀를 직접 확인.
