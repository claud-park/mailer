# label-crud — PRD

> 2026-07-16 · 오케스트레이터 도출 UX 개선 3건 중 2번. 사용자에게 두 번 반복 확인된 기존 백로그(`post-release 버그수정 3건`, `starred-view`의 "남은 스코프"). 브레인스토밍 설계 스펙: `docs/superpowers/specs/2026-07-16-ux-improvements-design.md`.

## 문제

사이드바는 라벨을 보고(list) 필터링(click)하고 스레드에 적용(l)할 수 있지만, 라벨 자체를 새로 만들거나 지울 방법이 앱 안에 전혀 없다 — Gmail 웹으로 나가야 한다. 키보드 중심으로 Gmail 탭을 완전히 대체하겠다는 앱의 목표(PRD.md 비전)에 정면으로 걸리는 구멍.

## 요구사항

### R1. 라벨 생성
- 사이드바 "Labels" 헤더 옆 `+` 버튼 → 인라인 이름 입력 → Enter로 생성.
- 색상은 Gmail 기본 자동 할당(사용자 선택 UI 없음).
- 생성 직후 목록에 즉시 반영(낙관적), 실패 시 롤백+토스트.

### R2. 라벨 삭제
- 각 라벨 행 hover 시 삭제 아이콘 노출 → 클릭 → **확인 다이얼로그**("'{name}' 라벨을 삭제하면 모든 메일에서 제거됩니다") → 확정 시 삭제.
- 그 라벨을 보고 있던 중이면 삭제 후 Inbox로 복귀.
- 실패 시 목록 무변경 + 실패 토스트.

## 논스코프
- 라벨 이름 변경(rename).
- 색상 직접 선택.
- 중첩(부모/자식) 라벨.

## 성공 기준
- 데모: `+`로 새 라벨 생성 → 사이드바에 즉시 표시 → 스레드에 적용 가능(l) 확인. 삭제 아이콘으로 라벨 제거 → 확인 다이얼로그 통과 후 사이드바·해당 라벨이 붙어있던 스레드 모두에서 사라짐.
- E2E `TC-LBL-*` 전건 PASS + 전체 스위트 무회귀.
