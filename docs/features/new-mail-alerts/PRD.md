# new-mail-alerts — PRD

> 2026-07-19 · 사용자 요청: "새로운 메일이 도착했을 때 app alert(badge + push alert) 기능 추가". 브레인스토밍 설계 스펙: `docs/superpowers/specs/2026-07-19-new-mail-alerts-design.md`.

## 문제

ZenMail은 앱을 열어야만 새 메일 도착을 알 수 있다(사이드바 in-app unread 배지조차 창을 봐야 보임). 백그라운드·다른 앱 사용 중에는 새 메일이 와도 인지할 방법이 전혀 없다. Dock 배지(전체 계정 unread 합산)와 네이티브 데스크톱 알림(발신자/제목)을 추가해 이 공백을 없앤다.

## 요구사항

### R1. Dock 배지 = 전체 계정 unread 합산
- 모든 로그인 계정(활성/비활성 무관)의 Inbox unread count를 합산해 `app.setBadgeCount()`로 macOS Dock에 표시.
- 로그인/데모 진입 직후 즉시 반영(기존 `refreshBadges()` 타이밍 재사용, 60s 데몬 틱을 기다리지 않음) + 이후 60s 데몬 틱마다 갱신.

### R2. 새 메일 판정 = Inbox(+Starred) unread 증가
- "새 메일"의 기준은 계정별 Inbox∪Starred(archive/trash/spam/스누즈 제외) unread 스레드 집합에 새로 추가된 ID — 단순 라벨 변경(읽음 처리, 보관, 라벨 추가/삭제 등 unread가 늘지 않는 변경)은 대상 아님.
- 계정 경계 없이 전역 합산(한 계정 신규분과 다른 계정 신규분을 합쳐서 판단).

### R3. 알림 내용 — 개별 vs 그룹
- 한 번의 판정 주기(≤60s)에 신규 unread 스레드가 정확히 1건이면: 발신자+제목을 그대로 노출하는 개별 알림.
- 2건 이상이면: "OO 외 N건" 형태로 그룹화한 알림 1개(전 계정 합산, 계정별로 나누어 여러 개 띄우지 않음).

### R4. 클릭 동작
- 개별 알림(1건): 앱 포커스 + 해당 스레드를 바로 오픈(그 스레드가 비활성 계정 소속이면 필요 시 계정 자동 전환 후 오픈).
- 그룹 알림(2건+): 앱 포커스 + **현재 활성 계정**의 Inbox로 이동만(계정 자동 전환 없음 — 여러 계정에 걸친 도착이어도 사용자가 있던 계정 그대로).

### R5. 포커스 중 억제
- ZenMail 창이 이미 포커스(frontmost) 상태면 알림을 띄우지 않는다(새 메일이 인박스에 바로 보이므로 중복·소음). Dock 배지는 포커스 여부와 무관하게 항상 최신 값으로 갱신.

### R6. 콜드스타트 안전
- 로그인/데모 진입 시점에 이미 존재하던 unread 메일 전체가 알림으로 한꺼번에 쏟아지면 안 된다 — 계정별 첫 상세 조회는 baseline 시딩으로만 취급하고 알림을 발화하지 않는다.

## 논스코프
- 알림 on/off 토글, 사운드 설정 — v1은 항상 ON(설정 UI 없음).
- Gmail Pub/Sub 기반 실시간 push — 별도 서버/웹훅 인프라 필요, 이 앱의 "백엔드 0" 아키텍처와 정면 충돌.
- 같은 60s 판정 주기 안에서 순증+순감소가 상쇄되는 경우의 정교한 이력 추적 — 기존 배지 시스템도 갖고 있던 60s 해상도 한계, 이번에 새로 풀지 않음.
- Windows/Linux 배지(오버레이 아이콘 등) — 이 프로젝트는 macOS DMG 전용 배포.

## 성공 기준
- 데모: 신규 unread 스레드 주입(디버그 훅) → dock 배지 증가 + 알림 발화(1건/그룹 각각) + 클릭 시 명시된 대로 이동, 전부 확인.
- 앱 포커스 중에는 배지만 갱신되고 알림은 안 뜸, 로그인 직후 알림 폭발 없음, 읽음/보관만으로는 알림 없음.
- E2E `TC-ALT-*` 전건 PASS + 전체 스위트 무회귀(0 FAIL + SKIP ⊆ 캐논 집합) ×2연속.
